使用Vite快读创建一个Vue项目 #

在上一个章节我们已经完成了开发环境的搭建，这一节我们就来介绍一下如何使用Vite来创建Vue项目。

一、Vite是什么？ #

Vite是一个前端的项目构建工具。它主要有两个作用，第一个作用是可以用来快速的创建一个前端项目，我们可以用它来对我们的项目进行创建和打包。第二个作用是可以作为一个服务器软件开运行我们的项目。Vite相对于其他工具来讲，最优秀的一个点就是运行的速度超级快，是其他工具的N倍！而且Vite创建的项目十分的简洁，对于强迫症来说真的是福音。 说了这么多，不过本质上Vite其实就是一个npm的包而已，所以使用起来很方便。

二、使用Vite来快速的创建项目 #

首先选择一个合适的文件夹用来作为我们存放项目的空间，后面我们创建的项目就可以直接在这个文件夹里面了。 假设这个文件夹的路径是 D:/workhome/Vite 打开cmd，进入这个文件夹

cmd
d:                   #切换盘符
cd D:/workhome/Vite  #进入刚刚选择的文件夹
1
2

然后通过npm的命令即可创建项目，我们只需要根据提示操作即可。一般就是让你输入项目的名称、选择项目框架等。

cmd
npm init @vitejs/app
1

如下图所示，即创建了一个叫viteDemo的Vue项目。

完成创建后，cmd中会显示我们如何启动我们的项目。 继续在cmd中输入命令，打开项目并且安装项目所需依赖。

cmd
cd viteDemo   # viteDemo是我们上一步创建的项目的名称
npm install   # 安装项目所需依赖
npm run dev   # 运行项目
1
2
3

运行成功后，如下图所示。 根据提示，我们就在在浏览器中输入提示的地址，去访问我们的项目了。 如果要停止运行项目，只需要选中cmd，在键盘上按Ctrl + C，终止项目运行即可。

三、Vite创建的项目结构说明 #

在cmd中通过命令使用VS打开当前项目，或者直接用VScode来打开项目。

cmd
code .
1

打开后，我们在VScode中可以看到如下目录。

- .vscode                 # VScode的配置文件，可以忽略不管
- node_modules            # npm安装的项目所依赖的第三方的包
- public                  # 项目的公共资源文件夹
- src                     # 项目的核心代码文件夹
    - assets              # 项目的静态资源文件夹
    - components          # 封装的组件
    - App.vue             # 项目的Vue入口文件
    - main.js             # 项目的核心文件
- .gitignore              # git的忽略文件配置
- index.html              # 访问项目的入口文件
- package-lock.json       # 项目所安装的依赖包的说明
- package.json            # 项目所需的依赖包说明
- README.md               # 项目介绍
- vite.config.js          # vite的配置文件

四、配置跨域 #

修改配置文件，在配置文件中新增server对象，对host、port、proxy进行配置。

export default defineConfig({
  plugins: [vue()],
  server:{
    port:3100,
    host:"0.0.0.0",
    proxy:{
      '/login': 'http://localhost:5000',
    }
  }
})
