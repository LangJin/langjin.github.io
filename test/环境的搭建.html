如果我们要测试某个软件， 那么我们就必须要先把这个软件给运行起来。 网站也是软件。 怎么利用tomcat来运行动态网站。

代码要去连接数据库，那么代码里面就包含了数据库的相关的连接的信息的。 jdbc://mysql://localhost:3306/testdb?user=root&password=123456

我给大家提供的软件的代码， 代码里面的内容是： jdbc://mysql://localhost:3306/db_morning?user=root&password=123456 也就是说，我们代码会使用root的账号，123456的密码去连接我们本机的数据库。 为了让代码能够成功的连上数据库，我们就必须把我们的数据库的密码修改成123456.

java写代码，最终交付出来的程序的文件的格式，要么jar格式，war格式。 war格式用的比较多。

步骤 #

  1. 修改数据库的账号root@localhost的密码为123456.
  2. 运行sql文件，创建所需要的数据库。
  3. 把代码放到webapps中
  4. 启动或者重新启动tomcat。
  5. 在浏览器中去访问我们运行的项目了。

搭建完成后，我们操作软件的功能，就会运行软件中对应的sql语句，来实现增加数据，修改数据，查看数据的功能。

在Linux上去搭建运行猫宁商城、还有birthday #

在Linux上利用tomcat来实现静态网站和动态网站的运行。

软件的所有的数据的状态，都是由数据库中的数据来决定的。

操作系统 #

  • Windows 我们平时用的最多的一个操作系统就是windows，但是在实际的工作中，我们一般很少会使用Windows的操作系统来作为服务器的操作。 Windows是商业的，所以使用Windows是要付费的。其次Windows的图形化页面是和系统绑定的。 所以如果使用Windows来作为服务器的操作系统的话，直接就会被系统占用系统电脑的资源。 所以只有很少的公司使用Windows来作为服务器的操作系统。
  • Linux Linux的操作系统是开源的，也就是说，Linux的使用是免费的。并且所有人都可以去下载Linux的代码，然后自己去修改，变成自己的操作系统。 所以其实市面上就有很多基于Linux实现的操作系统，比如：centos\kali\ubuntu\redhat\麒麟。 并且Linux的操作系统中，它的图形化页面这个功能是可以选择性安装的，也就是说，可以直接不要图形化的页面。 对于服务器来说，我们一般把软件程序在服务器上运行起来了后，就不会怎么再去动它了，所以图形化页面对于服务器来说没有意义。 如果没有图形化页面，还可以节约出来更多的电脑资源（CPU、内存、磁盘）给我们自己的程序使用。
  • OSX OSX是苹果电脑的操作系统。 大家都知道苹果电脑的操作系统是和苹果电脑绑定售卖的。 也就是说，如果我们要使用OSX这个操作系统，那么我们只能使用苹果电脑。 苹果电脑特别贵，所以对公司来说成本就很高，所以不会有公司使用OSX来作为服务器的操作系统。

环境 #

所谓环境在我们测试行业里面，指的就是软件的运行环境。

  • 开发环境 开发环境顾名思义，就是给开发人员使用的环境。开发写了代码，他们自己也要需要把他自己写的程序给运行起来去检查是否正确的。所以给开发人员运行代码的这个环境就是所谓的开发环境。开发环境上运行的软件的版本是最新的版本。
  • 测试环境 开发完成了对应的开发任务后，就会把写好的代码，交给测试。测试人员就需要在测试环境上去把这个软件给运行起来。运行起来了后，测试人员就在测试环境上进行测试，去操作这个软件。开发环境和测试环境是完全独立的，互不影响的。
  • 预生产环境 测试人员完成了软件的测试后，觉得没有问题了，在正式的交付给用户使用之前，需要先模拟一样在真实的环境上运行的效果，这个事情就需要在预生产环境上做。所以预生产环境上运行的代码和给用户使用的环境上运行的代码时一样的。预生产环境又叫做灰度环境，我们把软件放到预生产环境上去运行的这个过程，又叫做灰度发布。一般情况，α和β测试都是在预生产环境上进行的。
  • 生产环境 预生产环境上模拟了没有问题了后，才会正式放到生产环境上面去运行。生产环境就是真实的软件运行的环境，比如我们平时使用软件，微信、淘宝之类的，用的就是他们的生产环境。
