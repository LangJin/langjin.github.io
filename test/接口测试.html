接口测试和功能测试他们的测试流程是一模一样的，没有什么区别的。

认识接口 #

接口就是软件的不同的模块之间互相发送数据的一个通道。 在实际的生活中，接口是非常常见的一个东西，比如USB接口，TYPE-C接口，HDMI接口等等。这些是物理上能够看得见，摸得着的接口。虽然软件的接口摸不到，但是他们的作用和物理上的接口其实区别不大的。而且软件的接口是能够看到的。

接口的组成 #

  • 请求
    1. 请求头 请求头里面会包含很多信息，这些信息大多数都是浏览器自己的信息，他的作用就是告诉后端，发送请求的软件的相关的信息，比如是什么设备，发送的数据是什么格式，token是什么等等。
    2. 请求地址 请求地址就是对应的接口地址。
    3. 请求类型/方法 不同的接口使用的请求的方法是不一样的，常规的方法就有GET、POST。
    4. 请求数据 接口发送的数据，如果没有发送数据，那就看不到这个。
  • 响应
    1. 响应头 后端返回的信息，告诉前端，返回的数据的格式，来源、等等内容。
    2. 状态码 所谓的状态码，其实是http协议的状态，不同的状态码有不同的含义，方便浏览器去做对应的处理。
    3. 响应数据 后端返回的数据。

抓包 #

通过抓包的手段，我们就可以看到接口。

抓包工具 #

  • fiddler fiddle是Windows上最流行的抓包工具。
  • charles charles是OSX上最流行的抓包工具。
  • HttpCanary HttpCanary(小黄鸟)是Android上最流行的抓包工具。
  • network network是浏览器自带的一个抓包工具。

浏览器开发者工具 #

浏览器开发者工具是浏览器自带的一个功能，主要就是给开发者使用的，里面包含了很多各种的小工具。其中有一些功能我们就用的非常的多。 浏览器开发者工具是争对于某个具体的网站来用的，所以你要在哪个网站上使用浏览器开发者工具，你就必须先打开这个网站。然后可以直接按F12打开、或者在设置中，点击开发者工具打开。 这里使用浏览器开发者工具，推荐使用chrome/edge浏览器。

  • elements/元素 元素这里所显示的内容就是当前正在访问这个网页的代码。 前端程序员可以使用直接在这里修改代码的方式来调试代码的效果。

    • 审查元素 对于测试人员来说，我们常用的快速定位元素的功能，来实现快速的找到我们要的元素的信息，一般我们做UI自动化测试就用的很多了。 我们也可以快速的选中元素后，去修改他的内容，来测试数据展示的效果。
    • 模拟显示 我们可以利用浏览器来模拟这个网站在不同尺寸的设备上浏览的展示效果。 比如测试这个网站在手机上的浏览效果。

  • console/控制台 前端程序员可以直接在控制台这里允许JavaScript的代码。 并且在这里还会打印前端的日志。所以我们测试人员可以利用这个功能，来判断我们发现的BUG是什么原因导致的。

  • application/应用 这里所提供的功能就是用来存储前端的数据的。 比如之前我们学到的cookies就是在这里面的。 我们在测试的时候，我们就会去修改、删除里面的数据来观察软件的表现是否合理的，用来快速的模拟一下特殊的情况。

    • 本地存储 存储普通的数据
    • 会话存储 存储普通的数据
    • cookies token之类的数据就会存在这里

  • network/网络 浏览器自带的抓包工具。 我们可以通过抓包，去看到接口的信息。从而来判断接口有没有问题。 抓包查看接口，是我们用的最频繁的一个用来判断BUG是什么原因导致的技巧。

    • 抓包的原理 包指的是数据包，在网络上发送的数据都是以包为单位在发送，所以抓包指的就是抓发送的数据包，所以如果要成功的抓包，就必须要有一个利用接口发送数据的过程。
    • 接口的信息 当我们抓包成功后，我们就可以在列表中，点击我们想要看的，就可以看到对应的信息了。

日志 #

不管是前端的日志还是后端的日志，都是分了4个级别的。

  • debug 代码调试的时候的日志，我们一般都会忽略不管。
  • info 代码正常输出的信息，一般我们也不管。
  • waraing 代码的警告，比如代码编写的不规范，或者其他的提示信息，警告一般我们也不管。
  • error 代码的错误信息，这个我们需要重点关注的，所有的程序都是不允许出现error级别的日志的。出现了这个日志，就说明程序有问题。
