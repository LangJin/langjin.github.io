表的管理 #

对于数据库中的表而言，存在两种，一种真实的表，一种临时表。 他们两个表只有创建的sql语句不同，修改、删除之类的sql语句都是一样的。 比如：

sql
create table 表名 ();   -- 创建的真实表
create temporary table 表名(); --创建临时表
1
2

新建表 #

新建表的时候，重点就在于对字段属性的描述。 字段属性：

关键字说明
字段名必须要设置的
字段类型必须要设置的
字段长度必须要设置的
not null是否必填，根据情况是否设置
comment注释
default设置默认值，根据情况设置
auto_increment设置自动递增，根据实际的情况设置，要设置自动递增，必须要有主键的属性
primary key主键，约束这个字段的数据只能是唯一的，不能出现重复的数据。
index索引，利用索引可以提高对这个字段查询的速度。

例子：

sql
create table 表名(
    id int(16) not null comment "编号" auto_increment primary key,
    sname varchar(25) not null comment "姓名",
    sex varchar(12) default '保密'
);

限制查询结果的数量 #

在我们实际的工作中，数据库中的数据是很多的，动辄就是上万条，甚至几十万条数据，所以我们在查询的时候，不做任何限制的话，那么加载查询的结果的过程所花费时间是很夸张的。 所以我们一般都会限制查询的结果的数量。

sql
select * from 表名 limit x,y;
1
  • x,表示从第几条数据开始限制
  • y,表示一共显示多少条数据。
sql
select * from t_student limit 0,3;
1

聚合函数 #

聚合函数是mysql提供的用来对数据做统计计算的方法。

  • sum() 对你选中的字段的数据进行求和,null不会纳入计算的范围。
    • 语法：
    sql
    select sum(字段) from 表名;
select sum(字段) from 表名 where 查询条件;
    1
    2
    • 例子：
    sql
    select sum(age) from t_student;
select sum(age) from t_student where sex = '男';
    1
    2
  • avg() 对选中的字段的数据进行平均值的计算。

注意：如果数据中有null，那么null是不对纳入的计算的。

  • 语法：
sql
select avg(字段) from 表名;
select avg(字段) from 表名 where 查询条件;
1
2
  • 例子：
sql
select avg(age) from t_student;
select avg(age) from t_student where sex = '男';
1
2
  • count() 计算选择的字段中一共有多少条数据，null不会纳入计算的范围。

注意：因为count会忽略null的这种情况，所以我们想要计算一共有多少条数据的时候，会算不准。所以如果要计算一共（包括null）有多少条的时候，我们要么选择非空字段来进行计算，要么就直接写一个不存在的字段,或者直接用*也可以。

  • 语法：
sql
select count(字段) from 表名;
select count(字段) from 表名 where 查询条件;
1
2
  • 例子：
sql
select count(age) from t_student;
select count(age) from t_student where sex = '男';
1
2
  • max()
    • 语法：
    sql

    1
    2

分组 #

分组的意思，就是把我们的表里面的数据根据不同的条件分成不同的组。 然后就可以使用聚合函数对不同的组的数据分别进行计算。

注意：在标准写法里面，select后面写的单独的字段名，只能写group by后面用的分组的条件的字段，分组条件也是可以有多个的。

sql
select 字段,聚合函数 from 表名 group by 字段;
select 字段1,字段2,聚合函数 from 表名 group by 字段1,字段2;
1
2
sql
select sex,avg(age) from t_student group by sex;
select address,sex,avg(age) from t_student group by address,sex;
1
2

使用分组后查询出来的结果，如果我们要对结果进行查询，是不能使用where来进行搜索的。 where只能用于分组前的数据做查询，对分组后的数据，需要使用having来进行搜索查询。

sql
select 字段,聚合函数 from 表名 group by 字段 having 查询条件;
1
sql
select address,sex,avg(age) from t_student group by address,sex having avg(age) > 30;
1

表的类型 #

  • 真实的表 我们通过create table 这个语句所创建的表就是真实在数据库中，在文件中存在的一个表。
  • 虚拟表 虚拟表就是我们哦那个给select语句做查询，查询的出来的结果，就是虚拟表。 我们可以利用嵌套查询的方式来操作虚拟表。
  • 临时表 临时表的作用和真实的表是一模一样的。 但是临时表不可见，无法通过show tables看到临时表的存在。 并且一旦断开了数据库的连接，临时表就会自动销毁。 临时表是一次性的。 当我们要处理一些比较复杂的数据的时候，我们就先把查询出来的结果存放到临时表里面去，方便我们后续直接拿来用。

数据库的账号设置 #

以下内容是基于mysql5.7的版本来说的。

数据库的账号信息，是存储在mysql数据库的user表里面的。 数据库的账号的格式是：账号@IP地址。 账号就是我们连接的时候使用的账号。 IP地址的作用是说，你这个账号只能在数据库指定的IP的电脑上使用，不能在其他地方用。 所以，我们自带的这个账号root@localhost，就只能在数据库所在的电脑上使用，是不可以在其他的地方使用的。 如果不需要做IP的限制，那么你可以使用%，也就是账号@'%',这种写法的意思就是说这个账号没有IP的限制，可以在任意的IP地址上使用。

创建账号 #

sql
create user '账号'@'IP地址' identified with mysql_native_password by '密码';
grant all privileges on *.* to '账号'@'IP地址' with grant option;
flush privileges;

sql语句的分类 #

  • DDL data definition language DDL语句就是指的对数据库进行定义的语句，比如：create\alter\drop等等 简单的理解就是操作数据库、表、账号等等的语句。
  • DCL Data Control Language 数据库的功能性语句。 比如：begin\commit\rollback\set
  • DML Data Manipulation Language 操作数据的语句，比如：insert\delete\update等等
  • DQL Data Query Language 查询相关的语句。 select\from\where等等
