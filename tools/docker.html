docker概述 #

Docker 是一个开源的应用容器引擎，让开发者可以打包他们的应用以及依赖包到一个可移植的容器中,然后发布到任何流行的Linux或Windows操作系统的机器上,也可以实现虚拟化,容器是完全使用沙箱机制,相互之间不会有任何接口。 docker镜像资源：https://hub.docker.com/ docker文档介绍：https://docs.docker.com/desktop/ 一个完整的Docker有以下几个部分组成：

  1. DockerClient客户端
  2. Docker Daemon守护进程
  3. Docker Image镜像
  4. DockerContainer容器

docker的安装和卸载 #

docker可以在Windows和Linux以及OSX上运行。这里以Linux服务器centOS为例子进行讲解。

docker的安装 #

shell
# 需要的第三方依赖
yum install -y yum-utils 
# 设置镜像地址,推荐阿里云的地址
yum-config-manager --add-repo http://mirrors.aliyun.com/docker-ce/linux/centos/docker-ce.repo 
# 更新yum的索引
yum makecache fast  
# 安装,ce是社区版
yum install docker-ce docker-ce-cli containerd.io docker-compose-plugin
# 查看版本
docker version

docker的镜像操作 #

docker search 镜像名称          # 搜索线上资源镜像是否存在
docker pull 镜像名称:版本号     # 通过docker安装镜像，nginx:5.1表示安装5.1版本的nginx的镜像。如果不输入版本号，则默认安装最新的
docker images                   # 查看已经安装了的镜像
docker run 镜像名称             # 运行镜像，如果镜像不存在，会自动在搜索并下载
docker rmi -f 镜像ID            # 删除对应的镜像
docker rmi $(docker images -aq) # 删除所有镜像

docker的容器操作 #

说明：先有镜像才能创建容器。 通过docker pull命令下载的东西就是镜像，镜像启动后则称为容器。一个镜像可以同时运行很多次，每次运行就是一个新的容器。

docker run [可选参数] 镜像名称

  • 后台运行容器
docker run -d --name="nginx01" -p 3344:80 nginx
# -d 后台方式运行，如果容器启动后，没有任何的客户端进行访问，那么容器会自动停止。
# -it 交互方式运行
# --name 给容器取名字
# -p 宿主机端口:容器端口，把宿主机的端口和容器的端口进行映射
# -v 宿主机目录:容器目录，把宿主机的目录和容器的目录进行映射。

docker的数据同步 #

所谓的数据同步指的是，让docker容器里面的数据和宿主机的数据同步，所采用的技术叫做数据卷，docker可以实现容器内容的目录数据同步到宿主机的目录中，并且多个容器之间可以实现继承，继承后都可以和宿主机的目录同步，并且容器的状态不会影响宿主机的数据。宿主机和容器中的目录的映射关系就有点类似vue的双向绑定的关系。

指定挂载 #

docker run -it -v /home/langjin:/home nginx /bin/bash
# 通过这个操作，就可以让nginx这个镜像的容器和宿主机的目录进行同步
docker inspect 容器ID
# 通过这个命令就可以看，容器中的目录的映射。
# Mounts中显示的就是映射的目录。

docker commit -a="langjin" -m="在当前容器的基础上生成新的镜像" 23423rfsefe nginx:1.0
# docker commit -a="作者名称" -m="说明信息" 容器ID 新镜像名称:版本

方式二：DockerFile #

从无到有，手动制作一个新的镜像

sh
FROM centos:7.3.1611
MAINTAINER "zisefeizhu <zisefeizhu@zhujingxing.com>"
ENV nginx_ver=1.14.0
ENV nginx_url=http://nginx.org/download/nginx-${nginx_ver}.tar.gz
WORKDIR "/usr/local/src"
ADD CentOS-Base.repo  /etc/yum.repos.d/
ADD ${nginx_url} /usr/local/src/
RUN tar xf nginx-${nginx_ver}.tar.gz && \
        yum -y install gcc pcre-devel openssl-devel make &&  \
        cd nginx-${nginx_ver} && \
        ./configure && make && make install

语法说明

FROM                    #基础镜镜像，一切从这里开始构建
MAINTAINER              #镜像是谁写的，姓名+邮箱
RUN                     #镜像构建的时候需要运行的命令
ADD                     #步骤，tomcat镜像，这个tomcat压缩包! 添加内容
WORKDIR                 #镜像的工作目录
VOLUME                  #挂载的目录
EXPOSE                  #保留端口配置
CMD                     #指定这个容器启动的时候要运行的命令,只有最后一个会生效，可被替代
ENTRYPOINT              #指定这个容器启动的时候要运行的命令，可以追加命令
ONBUILD                 #当构建一个被继承DockerFile 这个时候就会运行ONBUILD 的指令。蝕发指令。
COPY                    #类似ADD，将我们文件拷贝到镜像中
ENV                     #构建的时候设置环境变量!
