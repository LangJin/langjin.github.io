云服务器 #

所谓云服务器，就是正常的服务器，没有什么很特殊的点。 那为什么我们要把服务器叫做云服务器呢，因为这里我们即将要使用的服务器没有在你身边。 我们只能通过网络远程去操作这个服务器。 这种服务器对于我们来说就是云服务器。

云：所谓云就是网络的意思。云盘、云原生、云。。。这些东西其实就是指的对应的内容是需要通过操作去操作的。

  • 腾讯云 领取云服务： 1、注册对应的账号，完成实名认证。 2、在首页、找到最新活动，在最新活动中找到你要的云产品，然后直接申请即可。 首页->最新活动->免费产品体验->个人专区->轻量应用服务器
  • 阿里云 1、注册对应的账号，完成实名认证。 2、在首页、找到最新活动，在最新活动中找到你要的云产品，然后直接申请即可。 首页->最新活动->试用中心->云服务器 2核4G
    • 重装系统 我们购买的服务器，操作系统可能不是我们要求的，所以我们可以通过重装系统的操作，去修改操作系统。或者在后续的操作中，你操作失误，导致了服务器无法正常使用了，而且你还不知道怎么去处理，这种情况，直接简单粗暴的重装系统就完事。 系统选择CentOs7.6
    • 启动 对服务器进行开机
    • 重启 对服务器进行重启的操作
    • 停止 对服务器进行关机
    • 设置密码,重置实例密码 这里设置的密码就是服务器的开机密码，因为服务器，如果你的密码太简单了，很容易被黑客攻击，所以这里密码一定要设置的复杂。 密码生成器：https://c.runoob.com/front-end/686/
    • 安全组/防火墙 安全组和防火墙是同一个东西。 在云服务器的控制台这里设置的防火墙不是服务器本身的防火墙。这个防火墙是由阿里云额外的提供的一个安全防护的功能。也就是说，我们对于云服务器里面的内容的访问，必须要先通过控制台这里的这个防火墙，然后再通过服务器里面的防火墙，才能正常的访问。 我们需要把我们在服务器运行的程序对应的端口号，添加到防火墙里面才可以。
    • 连接 我们可以利用控制台提供的连接的功能，远程去操作自己的服务器。 这个功能其实用的很少，因为要用它，每次都要打开浏览器，登录账号啥的，很麻烦。所以我们一般都用专门的连接工具去进行连接。 比如：向日葵就是一个典型远程连接工具，Windows上用的。 我们要连接我们的服务器，必须要知道服务器的IP、端口、账号、密码才可以。

远程连接工具 #

Linux大多都是作为服务器的操作系统来使用的。 服务器这个东西一天24小时一直开机不会关机，所以服务器这东西也不会在你的身边。 有的公司是自己的服务器，在自己公司的机房里面的。有的公司是直接用的云服务器，服务器在别人家的机房里面。不管是在自己公司还是别人的家里，我们都只能通过网络，利用远程控制软件去操作。

  • Xshell 这个软件就类似于向日葵这样子的远程控制软件，我们可以利用这个软件直接去操作服务器。 xshell只是一个远程控制服务器的工具。 我们是利用的xshell这个软件去操作的服务器！！！
  • Xftp XFTP是一个远程文件管理软件，我们可以利用这个软件去管理服务器上的文件，或者上传下载文件之类的操作。

注意：xftp和xshell的默认编码不一致，所以通过xftp上传的文件会出现乱码，所以需要修改xftp的编码为UTF-8

  • ll 查看当前目录下的文件,以详细信息的形式展示当前目录下的文件。 在Linux上做操作，Linux不会实时的展示你操作的结果，所以当我们做了任何对文件的操作，都可以习惯性的先输入一下ll，去查看你操作的结果。

  • cd 操作路径，简单的来说打开或者进入某个文件夹。 比如，我们想要打开某个文件夹，cd 路径 不管是什么操作系统，大家都要知道什么是路径。 关于路径，有两种路径，一种叫 绝对路径，一种叫相对路径。 绝对路径和相对路径的作用是一模一样的，只是表达的方式不同而已。 绝对路径：就是一个完整的路径。以Windows系统为例：F:\root\日记本，Linux上的绝对路径和Windows是一样的表达方式。唯一不同的就是Linux的路径最前面没有盘符。\root\日记本 相对路径：相对路径是一种更加简洁的路径的表达方式，它是根据你现在所在的路径作为参考来表达路径的一种方式。在相对路径中我们经常会用到一些特殊的符号来表示对应的含义。 比如：
符号说明
.当前所在的路径
..表示上级路径
-表示上一次访问的路径
～表示用户路径,root这个账号是很特殊的，它是超级管理员的账号，所以用root账号登录的时候，用户路径就是/root，如果是其他的账号登录的，比如zhangsan，那它的用户路径是/home/zhangsan
/表示根路径
如果这个路径是/开头的，那肯定是绝对路径的写法。如果前面不是/那肯定就是相对的路径的写法。

Linux的文件读写 #

vi #

不管是什么操作系统，对文件进行读写本质上都是通过了软件来进行的。 比如在Windows上，我们对txt的文件是通过记事本软件来进行读写的，对于doc、xls等文件是通过了office、wps等软件进行读写的。 在Linux上也是一样的。 不过因为Linux主要是作为了服务器的操作系统来用。所以我们Linux上的文件大多数都是文本类型（记事本）的文件。所以我们都可以直接使用Linux版本的编辑软件（类似于记事本）来进行操作。 Linux自带的编辑软件叫 【vi】。（vi有个高级版叫vim） 所以Linux上对文件的读写，主要就是学习vi这个软件的使用。

  • vi的模式说明 对于操作vi这个软件来说，有两种常见的模式，分别是 命令模式，编辑模式。 在命名模式下面，我们就可以进行保存、退出之类的操作。 在编辑模式下面，我们就可以编辑文件的内容。

注意：所有的指令都需要在英文输入法下面去按才有效。

指令说明
i/a/insert在命令模式下面，按这三个按键都可以进入编辑模式。
esc在任意模式下面，按ESC键，都可以进入命令模式。
:w在命令模式下输入:w就是保存的意思，如果是直接vi进来的，那么我们在:w后面还需要加上文件名才可以保存。
:wq保存并退出。如果是直接vi进来的，那么我们在:w后面还需要加上文件名才可以保存。
:q退出，如果你做了任何的修改，都是无法直接退出的
:q!不保存退出的意思。
  • 使用vi来创建新文件。 直接输入vi就是打开了vi这个软件，然后我们就可以通过vi来创建一个新的文件

【MySQL的离线rpm包安装步骤】 1、通过工具，连接上Linux服务器，然后将我们准备好的mysql的安装包传输到服务器中。 2、对mysql进行解压，命令是 tar xf 文件名 3、安装numactl（必要组件，不安装会导致后面的步骤出现依赖的问题。） yum -y install numactl 3.1、卸载mariadb（这是系统自带的数据库，不卸载会导致MySQL安装失败。） rpm -qa | grep -i mariadb rpm -e --nodeps mariadb-libs-5.5.60-1.el7_5.x86_64（这个文件名字是上一步查出来的） 3.2、安装mysql，按顺序安装下面4个rpm（版本号可能不一样）。 rpm -ivh mysql-community-common-5.7.23-1.el7.x86_64.rpm rpm -ivh mysql-community-libs-5.7.23-1.el7.x86_64.rpm rpm -ivh mysql-community-client-5.7.23-1.el7.x86_64.rpm rpm -ivh mysql-community-server-5.7.23-1.el7.x86_64.rpm 4、等待安装结束后，启动数据库。命令：systemctl start mysqld.service 5、检查数据库是否运行成功，命令：systemctl status mysqld.service 6、数据库安装成功后，先生成一个默认密码，查看密码的命令：cat /var/log/mysqld.log | grep password 7、使用上一步获取的密码连接数据库，mysql -u root -p 这个步骤你们肯定没问题的。 8、进入数据库后，必须修改密码才能做其他的操作，所以修改密码为1qaz!QAZ，命令：ALTER USER 'root'@'localhost' IDENTIFIED WITH mysql_native_password BY '1qaz!QAZ'; 9、然后退出数据库，用新密码重新登录。 10、然后创建一个具有远程访问权限的账号。有3条语句，如下： create user 'root'@'%' identified with mysql_native_password by '1qaz!QAZ';
grant all privileges on . to 'root'@'%' with grant option; flush privileges; 10.1、为了让数据库的密码能修改为123456，所以我们需要对数据库进行一些配置。命令如下，完成后，就可以通过navicat来随意修改密码了。 SHOW VARIABLES LIKE 'validate_password%'; 查看数据库的密码规则 set global validate_password_policy=LOW; 修改密码强度要求 set global validate_password_length=6; 修改密码长度要求 11、好了数据库的设置结束了，你现在可以尝试能不能用navicat来连接了。 12、如果不能，那八成是端口的问题。所以检查阿里云的控制台的安全组是否开放端口。 13、通过命令查看当前已经开放的端口：netstat -ntlp 14、如果不存在3306，那么通过以下2个命令打开3306端口号。 将3306端口添加到防火墙例外并重启： firewall-cmd --zone=public --add-port=3306/tcp --permanent firewall-cmd --reload 15、再次尝试navicat能连接了不。

shell脚本 #

shell就是Linux操作系统的批处理脚本。 和Windows的批处理脚本是一个性质的东西。 所谓shell脚本，就是像写代码一样的方式，来把Linux的简单的命名组合起来，变成一个复杂的操作。然后有了这个shell脚本后，对于经常会去操作的复杂操作，我们就可以每次只运行一下这个shell脚本就可以实现了。

sh格式的文件就是shell脚本。

每天早上，我们都需要连接上服务期，然后查看某个目录下的文件，分别有哪些，并且记录下来。

sh
echo "执行停止tomcat的命令。。。。"
sh /usr/apache-tomcat-8.5.43/bin/shutdown.sh
echo "tomcat已停止。。。"
echo "执行启动tomcat的命令"
sh /usr/apache-tomcat-8.5.43/bin/startup.sh
echo "tomcat已启动"
echo "tomcat已完成重启"
