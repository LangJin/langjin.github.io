路由 一个网站要实现不同的页面的跳转，其实就是通过了路由来实现的。 所谓路由简单的来理解就是网址里面所写路径。这个路径在我们前端的专业术语中就叫做路由。 不同路由对应不同的页面，然后通过不同的路由之间切换从而实现了页面的跳转的过程。

项目结构 #

要创建一个多页面的项目，那么我们肯定是有很多个页面的。 一般情况，一个页面就是一个vue文件。 我们一般会把我们项目中所需要展示的页面全部都放到一个单独文件夹中进行管理。 比如:view,pages等等这样子的名字。

创建页面 #

你需要多少个页面，然后去编写页面的对应的vue的文件就可以了。 首先主页，是肯定需要的，所谓的主页就是我们进入这个项目默认访问的页面。约定俗称，这个页面的名字最好叫index.vue。当然这个要求不是强制性。

创建路由 #

在vue里面，创建路由使用的是vue-router这个第三方的包来实现的。 vue-router这个包也是vue官方指定的做路由管理的包。

地址：https://router.vuejs.org/zh/

路由模式 #

方法模式说明
createWebHashHistoryhash哈希模式会自动的处理资源和路径的关系，当部署到服务器里面的时候，不会因为tomcat或者nginx的原因，导致资源的丢失。但是哈希模式会在路径前面显示#号，很难看。
createWebHistoryhistory这个模式就不会显示#,但是用web 服务器软件来运行的时候，在访问路由的时候，服务器软件就会找不到对应的资源，需要在nginx上面做额外的设置。

对路由进行封装 #

js
import {createRouter,createWebHashHistory} from 'vue-router'
// createRouter 这个是vue-router里面用来创建路由的方法。我们要实现路由的创建就必须使用它来完成。
// createWebHashHistory 路由模式的哈希模式。
// createWebHistory    路由模式的历史模式。
import home from '../view/home'
const routes = [//绑定路径和vue文件的关系
    {
        path:"/",
        component:home
    },
    {
        path:"/login",
        component: () => import('../view/login.vue')
    },
    {
        path:"/my:name",    //:name是一个参数，这是动态路由的写法，比如name=张三，那么这个路由就是/my/张三
        component:() => import('../view/my.vue')
    },
    {
        path:"/:pathMatch(.*)*",    //404的路由写法
        component:() => import('../view/404.vue')
    }
]

const router = createRouter({   // 创建路由
    history:createWebHashHistory(), //使用hash模式
    routes:routes   //加载路由
})

export default router;  //导出路由

在项目中加载路由信息 #

一般来说，只要是第三方的东西，我们要在项目中以全局操作的方式去使用，那么我们都是需要把对应的东西给加载到项目中。

js
import { createApp } from 'vue'
import './style.css'
import ElementPlus from 'element-plus'
import 'element-plus/dist/index.css'
import router from './router'
import App from './App.vue'

const app = createApp(App)
app.use(ElementPlus)
app.use(router)
app.mount('#app')

路由守卫 #

我们可以根据具体的条件来判断，是否允许访问某些页面。 比如，在没登录的时候，就不允许访问个人中。

js
router.beforeEach((to,from,next) =>{    // to要访问的路由的属性，from是当前的路由的属性，next类似于push()的作用/
    let token = localStorage.getItem("token")
    if (!token && to.path == "/my"){
        next("/login")
    }else{
        next()
    }
})

操作路由 #

  • 跳转页面
js
this.$router.push("/")  // 跳转到某个路由
this.$router.go()   // 这个写法的作用是等同于history的方法的使用的。
this.$router.replace("/")   // 替换掉当前的路由
1
2
3
  • 传递参数
js
//这两种写法的效果是一样的，第一种是直接在路由后面拼接来实现参数的传递。
//第二种是通过在push（）里面传参来实现的。
this.$router.push("/?username="+this.username) 
this.$router.push({path:"/",query:{name:this.username}})
//动态路由的参数传递
this.$router.push("/my/"+this.username) 
this.$router.push({path:"/my",params:{name:"张三"}})

路由的属性 #

  • path 路由地址
  • query 路由的query类型的参数
  • params 路由的params类型的参数
  • name 路由名字
  • meta 路由的元属性，在元属性里面可以写很多参数
  • alias 路由的别名，写法和作用和path一样。
