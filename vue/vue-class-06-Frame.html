vue的框架的搭建 #

所谓的框架，就是自己根据项目的需求封装好对应的代码，然后分门别类的放到对应的文件夹里面去。 我们肯定是可以根据自己的想法自己去搭建完成一个框架的。 但是一般我们不这样子做，毕竟我们现在还是一个初学者，我们虽然可以搭建框架，但是所搭建出来的框架肯定会有很多不合理的地方，很多不成熟的想法。 所以一般我们都是直接使用已经非常成熟的框架去使用。

使用别人封装好的框架 #

  1. 找到他们对应的代码仓库
  2. 下载项目的代码
  3. 安装所依赖的第三方的包
  4. 启动
  5. 修改成自己要的效果。

package.json #

这个文件里面就描述我们的这个项目的基本信息。 包括了项目的名称、版本信息、作者信息、调试的脚本的命令、以及所需要的第三方的包的信息。 比如：

  • name,项目的名称
  • version,版本信息
  • author,作者信息
  • scripts,操作项目的命令的信息，重点，这里的命令都是可以自己去修改配置的。
    • dev,这个命名一般用于项目的启动
    • build，用于编译代码，打包
    • serve，一般也是用于项目的启动
  • dependencies,项目所依赖的包的信息，重点
  • devDependencies,开发的时候使用到的项目所依赖的包，重点
  • lint-staged,对代码的语法进行检查
  • config,对项目的一些设置，比如端口号
  • keywords,项目在搜索引擎中触发的关键字
  • repository,项目的仓库地址
  • license,开发协议
  • bugs,BUG表单
  • homepage,项目的主页

认识项目结构 #

vue.config.js这个东西和vite.config.js是同样性质的一个文件，因为我们之前的项目是使用vite来创建生成的项目结构。而这个项目使用的是vue-cli来生成的项目结构。所以他们的配置文件是不同的。

┌── public				# 静态资源(不会被Webpack打包)
├── src					# 源代码
│	├── api				# 接口
│	├── assets			# 静态资源(会被打包)
│	├── components		# 组件库
│	├── config			# 配置
│	├── directives		# 自定义指令
│	├── layout			# 框架布局视图
│	├── locales			# 国际化文本
│	├── router			# 路由表
│	├── store			# VUEX状态管理
│	├── style			# 全局样式
│	├── utils			# 全局公用方法
│	├── views			# 所有视图
│	├── App.vue			# 入口视图
│	└── main.js			# 入口文件
├── .editorconfig		# 统一编辑器样式
├── babel.config.js		# postcss 配置
├── package.json		# 包管理
└── vue.config.js		# vue-cli 配置
