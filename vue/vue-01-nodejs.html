Vue是一个基于nodejs的前端开发框架。所以要使用Vue进行前端开发之前，我们必须要先准备好nodejs的编程环境。

准备工作 #

在正式搭建环境之前，我们需要准备好nodejs的安装包，以及一个代码编辑器工具，比如VsCode。需要注释的是，在下载的时候，建议都选择稳定版进行下载。

nodejs下载地址：https://nodejs.org/zh-cn/
VsCode下载地址：https://code.visualstudio.com/

安装配置nodejs #

安装nodejs #

nodejs环境安装十分的简单，直接双击运行安装包即可，所有选项全部默认点击下一步，不需要做任何的额外的设置。 安装完成后，nodejs的环境变量也会自动的添加到系统中，不需要手动进行配置，十分方便。
nodejs的安装包中，包含了JavaScript的运行环境node，和JavaScript的第三方的包管理工具npm，他们的关系与python的环境和pip的关系基本一致。所以在安装结束后，我们需要检查nodejs和npm是否安装成功。
打开输入以下命令,如果安装成功，会出现对应的版本号。

注意：如果没有正常的显示，请检查环境变量是否正常，然后重新配置环境变量即可。

cmd
node -V  # 检查node的版本号
npm -v  # 检查npm的版本号
1
2

配置npm #

在我们通过npm来进行安装JavaScript的依赖包时，如果不做配置会自动的安装到C盘的用户目录下，当时间久了后，就会导致下载的第三方的包十分巨大且占用空间，而且不方便管理。所以我们需要自定义npm包的下载路径和缓存的路径。
首先自己在合适的位置创建2个文件夹，分别取名叫node_global和node_cache，当然，你也可以取其他的名字。然后将node_global的路径配置到环境变量PATH中。
在cmd中输入命令就可以修改npm的包的下载路径和缓存路径为你刚刚创建的两个文件夹了。

注意：路径切记要用双引号，单引号会引起其他问题。

cmd
npm config set prefix "D:\Program Files\nodejs\node_global"   # 路径是你创建的文件夹的路径
npm config set cache  "D:\Program Files\nodejs\node_cache"    # 路径是你创建的文件夹的路径
1
2

配置完成后，可以通过以下命令检查是否配置成功。

cmd
npm config get prefix  # 查看npm包的下载路径
npm config get cache   # 查看npm包的缓存路径
1
2

其他的包管理工具 #

前端的生态建设十分的混乱，有各种公司在对第三方的包进行管理和维护，不同的公司的仓库的第三方的包甚至还有可能不一样，所以我们还需要安装其他的第三方包管理工具。

cnpm #

cnpm是淘宝出的一个npm的包管理工具，因为npm所安装的包的都是在国外的，所以用npm进行安装第三方的包速度比较慢，所以我们可以通过安装cnpm来解决这个问题。
通过cnpm安装第三方的包的时候，会自动的使用淘宝的镜像，下载速度就更快，可以很方便的节约我们开发的时间。 在cmd中输入命令，即可安装cnpm。

cmd
npm install -g cnpm --registry=https://registry.npm.taobao.org
1

安装完成后，在cmd中输入 cnpm -v 检查cnpm是否安装成功。

cmd
cnpm -v
1

yarn #

yarn也是一个包管理工具，它是FaceBook出的一个包管理工具，本质上和npm没什么区别，只是有的人喜欢使用npm，有的人喜欢使用yarn而已。yarn的安装和cnpm的安装一样，也是直接一条命令就可以完成。

cmd
npm install -g yarn
1

安装完成后，在cmd中输入 yarn -v 检查yarn是否安装成功。

使用Vite创建Vue项目 #

Vite是一个前端的项目构建工具。
它主要有两个作用，第一个作用是可以用来快速的创建一个前端项目，我们可以用它来对我们的项目进行创建和打包。
第二个作用是可以作为一个服务器软件开运行我们的项目。
Vite相对于其他工具来讲，最优秀的一个点就是运行的速度超级快，是其他工具的N倍！而且Vite创建的项目十分的简洁，对于强迫症来说真的是福音。
说了这么多，不过本质上Vite其实就是一个npm的包而已，所以使用起来很方便。 参考资料：https://cn.vitejs.dev/guide/

cmd

cd D:/workhome/Vite  #进入刚刚选择的文件夹

用VsCode运行项目 #

VsCode的安装配置在这里就不讲了

运行VsCode，点击菜单栏的文件->打开文件夹，选择我们刚刚创建的项目vite-demo即可。 打开后，我们在VScode中可以看到如下目录。 这些文件和文件夹就是vite给我们初始化的内容，在后续的项目开发中，我们就只需要在vite帮我们创建好的目录的基础上不断补充即可。
使用VsCode打开项目后，直接在终端输入启动命令就可以把项目启动起来了。

相关目录解释：

- .vscode                 # VScode的配置文件，可以忽略不管
- node_modules            # npm安装的项目所依赖的第三方的包
- public                  # 项目的公共资源文件夹
- src                     # 项目的核心代码文件夹
    - assets              # 项目的静态资源文件夹
    - components          # 封装的组件
    - App.vue             # 项目的Vue入口文件
    - main.js             # 项目的核心文件
- .gitignore              # git的忽略文件配置
- index.html              # 访问项目的入口文件
- package-lock.json       # 项目所安装的依赖包的说明
- package.json            # 项目所需的依赖包说明
- README.md               # 项目介绍
- vite.config.js          # vite的配置文件

总结 #

Vue的开发环境的搭建的过程就是这样子的了。 在我们完成了nodejs的环境的安装后，使用vite就可以快速的完成一个项目的创建了。 项目创建完成后，我们通过npm命令正常的运行项目即可。 不过有时候项目会运行失败，这种情况一般都是项目的依赖没有正确安装导致的，我们只需要重新安装依赖，然后再次运行即可。
