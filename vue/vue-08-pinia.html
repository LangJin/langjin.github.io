在Vue中状态管理可以直接理解成全局变量的管理。 vue对状态管理的实现的方式很多。常见的有vuex\pinia等等。 现在主要流行的就是使用pinia来实现。

安装pinia #

npm install -s pinia
1

pinia介绍 #

pinia由三部分重要的数据组成，分别是state\getters\actions。

js
import { defineStore } from "pinia";
// 第一个参数是应用程序中 store 的唯一 id
export const useUsersStore = defineStore("users", {
  state: () => {
    return {
      name: "小猪课堂",
      age: 25,
      sex: "男",
    };
  },
  getters: {
    getAddAge: (state) => {
      return (num: number) => state.age + num;
    },
    getNameAndAge(): string {
      return this.name + this.getAddAge; // 调用其它getter
    },
  },
  actions: {
    saveName(name: string) {
      this.name = name;
    },
  },
});

  • state 用于存储公共变量的数据。和选项式API中的data一个作用。 在state中定义的变量如果和vue中的data变量同名，在使用的时候不会生效。
  • getters 用于处理一些简单的计算。
  • actions 用于定义一些逻辑处理的方法，同步、异步的方法都可以。

对状态进行定义 #

对于这种第三方的js包而言，使用的时候都应该先对他进行单独的导入和封装，然后再挂载到项目中。 首先在项目中创建一个叫[store]的文件夹，在里面创建一个叫index.js的文件,用于导入pinia的包。

js
import { createPinia } from "pinia";
const pinia = createPinia();
export default pinia;

然后再创建所需要的具体的全局状态的内容，写到[store]里面的一个叫user.js的文件夹中。

js
import {defineStore} from 'pinia'
export const useStore = defineStore('user',{
  state:()=>{
    return {
      //需要用到的各种变量或者对象
    }
  },
  getters:{
    //计算变量的各种方法
  },
  actions:{
    //处理状态的各种逻辑方法
  }
})

在项目中使用store #

在pinia中自带了各种操作store的方法，除了前面用到的defineStore用于创建Store对象的方法外，还有操作它的各种方法。

  • mapState 用于读取store中的state和getters中的值，在选项式API中，需要写到计算属性computed中。使用的时候直接通过this使用。
  • mapActions 用于访问store里面定义的action方法，在选项式API中，需要写到methods中。
  • mapWritableState 用于读取state中的变量，但是通过这个方法读取出来的数据是可以修改的。在选项式API中，需要写到计算属性computed中。使用的时候直接通过this使用。
vue
<script>
import {mapActions,mapState,mapWritableState} from 'pinia'
import {useStore} from '../store/user'
export default {
  data(){
    return{

    }
  },
  computed:{
    ...mapState(useStore,['state变量名字','getters方法的名字']) //用[]只能正常的读取store中的变量,变量名不变
    ...mapState(useStore,{  //用{}可以对store中的变量赋值给新的变量，但是内存地址不变。，可以定义新的getters方法
      自定义变量名:"state变量名字",   //读取state
      自定义变量名:"getters方法的名字", //读取getters
      自定义变量名:(store) => store.name, //语法糖
      自定义变量名(store){   //新方法
        return store.name
      }
    })
    ...mapWritableState(useStore,['state变量名字','getters方法的名字']) //用[]只能正常的读取store中的变量,变量名不变
    ...mapWritableState(useStore,{  //用{}可以对store中的变量赋值给新的变量，但是内存地址不变。不能定义方法
      自定义变量名:"state变量名字",   //读取state
      自定义变量名:"getters方法的名字" //读取getters
    })
  },
  monthods:{
    ...mapActions(useStore,['action方法名']),  //导入后就可以用this方法使用
    ...mapActions(useStore,{  //用{}可以取别名,导入后就可以当作普通的方法使用
      自定义变量名:'action方法名'
    })  
  }
}
</script>
